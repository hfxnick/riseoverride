Privacy Policy

Your privacy matters. This page explains how RiseOverride handles your data.

Last updated: June 24, 2025

At Override Labs, your privacy is important to us. This policy explains how we collect and use your data through the RiseOverride Chrome Extension and our website, overridelabs.ca.

1. What We Collect

We do not collect personal data by default. If you use Pro Mode, submit content, or request support, you may provide your email address. This is only used for licensing, communication, and support purposes.

2. Chrome Extension Permissions

  • downloads: Enables SCORM file interception and patching.
  • scripting: Injects a custom alert into publishing UI when automatic SCORM Patching is enabled (Pro users), and will be required to develop roadmap features (like live CSS previews).
  • storage: Stores your custom code locally (and in sync for Pro users).
  • alarms: Supports daily background license validation checks.
  • host_permissions: Detects SCORM downloads from Articulate servers for when Automatic SCORM Patching is enabled.

3. Pro Mode Licensing & Payments

Pro Mode requires license activation using an email address and product key. Licensing is handled through LemonSqueezy, a Stripe company. We do not process or store any payment information directly.

License status is validated once every 24 hours using a lightweight Cloudflare Worker proxy. Your license key and a machine-specific ID are transmitted only for validation purposes.

To prevent concurrent use, each Pro Mode license is limited to one active device at a time. If you attempt to activate your license on a different device, you may be asked to transfer or release the previous session.

For convenience, Pro user preferences and code are stored using chrome.storage.sync, which enables automatic syncing between Chrome instances. A snapshot of your license status is also temporarily stored here for performance.

4. Use of OpenAI API for Code Generation & Explanations

Pro Mode users have access to optional features that use OpenAI's API to generate code, provide explanations, or optimize code snippets. No data is sent to OpenAI unless you actively use these features by highlighting code and selecting Generate, Explain, or Optimize. Your code is only transmitted to OpenAI when you take these actions, and never automatically.

For more information about how OpenAI handles data, please see the OpenAI Privacy Policy.

5. Website Analytics

We use Cloudflare Web Analytics on our website (overridelabs.ca) to understand traffic patterns. This data is anonymized and used solely for internal insights. No analytics or telemetry exist within the Chrome Extension.

6. Block Library Submissions

If you submit a custom stylesheet to the Block Library, we collect your email address through a Google Form. This is used only to contact you about your submission and to grant free Pro Mode access if approved.

7. Policy Updates

We may update this policy as new features are released or as licensing systems evolve. Please check back regularly for the latest version.
